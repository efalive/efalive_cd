Erstellen eigener Layouts für die Ausgabe

Für alle HTML-basierten Ausgabearten (im Programm (Grafik), im Browser, als HTML-Datei), sowie für die Ausgabe als PDF-Datei und als XML-Datei erlaubt efa die Verwendung verschiedener Formatierungen (Layouts). Insbesondere kannst Du als Nutzer selbst Layouts erstellen, was es Dir ermöglicht, die Ausgabe völlig frei nach Deinen Wünschen zu gestalten. Es lassen sich sogar auf diese Weise noch weitere Daten berechnen, die efa normalerweise nicht ausgibt.

Die Verwendung von Layouts beruht auf einer Transformation einer von efa erstellten XML-Datei mittels XSLT (http://www.w3.org/Style/XSL/): Sobald nicht die Standard-Formatierung ausgewählt ist, erstellt efa in Wirklichkeit eine temporäre XML-Datei, die alle Daten enthält, und transformiert diese dann mittels eines XSLT-Stylesheets in die eigentliche Ausgabedatei.

Zum Schaffen eigener Formatierungen muß also ein solches XSLT-Stylesheet erstellt werden. Damit efa es anschließend als Formatierung für die Ausgabe anbietet, muß es im Verzeichnis ausgabe\layout in eines der drei Verzeichnisse html, pdf oder xml kopiert werden (je nachdem, ob es eine Formatierung für HTML, PDF oder XML vornimmt). efa verwendet den Dateinamen des Stylesheets dann als Name für die Formatierung.

Beim Erstellen eines eigenen Stylesheets können die mitgelieferten Stylesheets als Vorlage dienen. Besonders das Stylesheet XSLTDemo.xsl für die HTML-Ausgabe demonstriert einige Möglichkeiten, die sich mit XSLT ergeben. Ich werde mich bemühen, mit der Zeit weitere Stylesheets zu schreiben und die bestehenden zu erweitern.

Erstellen eines eigenen Stylesheets

Zum Erstellen von eigenen Stylesheets gibt es seit der Version 1.1.0 das Programm elwiz - efa Layout Wizard (ist Bestandteil von efa). elwiz ermöglicht es, mit wenigen Mausklicks ein neues Stylesheet zu erstellen. Die gestalterischen Möglichkeiten sind dabei naturgemäß beschränkt; um sämtliche Möglichkeiten voll auszuschöpfen, müßtest Du ein Stylesheet von Hand schreiben. Dabei wirst Du wohl um Grundkenntnisse in XSLT nicht herum kommen. Eine Vielzahl von Links zu diesem Thema findest Du unter http://www.w3.org/Style/XSL/.
