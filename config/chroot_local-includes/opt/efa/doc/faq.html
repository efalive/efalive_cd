Antworten

  • Frage 1: Ich möchte efa gerne direkt im Bootshaus einsetzen, so daß die Mitglieder ihre Fahrten selbst eintragen können. Unterstützt efa solche Funktionen?
    Antwort: Ja! Seit Version 1.3.0 ist efa für den Einsatz im Bootshaus gerüstet! Weitere Informationen darüber findest Du hier.

  • Frage 2: Müssen bei der Eingabe von Fahrten alle Felder ausgefüllt werden?
    Antwort: Nein, nicht unbedingt. Um die Bedeutung der einzelnen Felder für efa und die Auswertung vollständig zu verstehen, lies bitte das Kapitel 3 der Tour aufmerksam durch. Dort wird auch diese Frage beantwortet. Du kannst die Tour über den Menüpunkt Tour im Menü Info starten, oder Dir das Kapitel direkt hier im Browser anschauen.

  • Frage 3: Ich werde beim Starten der Bootshaus-Version immer nach einem Admin-Namen und einem Paßwort gefragt. Was muß ich hier eingeben?
    Antwort: Der Admin-Name des Super-Administrators (der Administrator, der alles darf) lautet admin. Das Paßwort hast Du selbst gewählt. Sollteset Du dieses Paßwort vergessen haben, so wende Dich bitte per email an software@nmichael.de, um Hilfe zum Reaktivieren von efa zu erhalten. (Aus Sicherheitsgründen gibt es keine Anleitung zum Reaktivieren von efa in der Online-Hilfe!)

  • Frage 4: Wenn ich unter Windows im efa-Verzeichnis auf das Icon mit dem Ruderer klicke, startet efa nicht, sondern es erscheint ein Öffnen mit...-Dialog. Was mache ich falsch?
    Antwort: Die Datei mit dem Ruderer als Symbol ist nicht die Programmdatei, sondern nur das Icon (Bild). Um efa zu starten, mußt Du die Batchdatei efa.bat (bzw. efadirekt.bat für die Bootshaus-Version) anklicken. Diese Datei hat als Symbol ein Konsolenfenster mit einem Zahnrad. Je nach Einstellung von Windows wird u.U. die Extension .bat nicht angezeigt, sondern nur efa bzw. efadirekt.

  • Frage 5: Ich möchte eine neue Version von efa installieren. Wie gehe ich vor?
    Antwort: Wenn der Computer über einen Internet-Zugang verfügt, dann empfielt es sich, die von efa bereitgestellte Funktion zum Online-Update zu nutzen: Hierbei prüft efa, ob eine neue Version verfügbar ist, läd diese herunter, sichert alle eigenen Daten und installiert die neue Version. Das efa Online-Update wird im efa-Hautfenster im Menü Datei über den Punkt efa Online-Update gestartet.
    Vor einer manuellen Neuinstallation empfehle ich, vorsichtshalber immer die Daten zu sichern (auch wenn dies eigentlich nicht nötig sein sollte). Kopiere dazu alle Dateien aus dem \efa\daten-Verzeichnis an einen anderen Ort. Anschließend kann eine neue Version von efa in dasselbe Verzeichnis, in dem sich auch die alte Version befindet, installiert werden, ohne daß die alte Version zuvor deinstalliert werden muß. Die Einstellungen und Daten bleiben dabei erhalten.

  • Frage 6: Nachdem ich efa-Daten von CD-ROM auf die Festplatte kopiert habe, meldet efa, daß Dateien nicht erstellt werden können oder schreibgeschützt sind. Was ist falsch?
    Antwort: Da eine CD-ROM nicht schreibbar ist, sondern Dateien von CD nur gelesen werden können, setzt Windows für alle Dateien und Verzeichnisse auf einer CD-ROM das Attribut "schreibgeschützt". Beim Kopieren der Dateien von einer CD-ROM zurück auf die Festplatte bleibt dieses Attribut erhalten, d.h. alle kopierten Verzeichnisse und Dateien sind schreibgeschützt. Windows erlaubt es efa daher nicht, die Dateien zu verändern oder in schreibgeschützten Verzeichnissen neue Dateien anzulegen.
    Um das Problem zu beheben, entferne den Schreibschutz von allen Dateien und Verzeichnissen. Um dies zu tun, kannst Du im Windows-Explorer alle betroffenen Dateien und Verzeichnisse markieren, nach einem Rechtsklick mit der Maus aus dem Kontextmenü den Punkt Eigenschaften auswählen und anschließend die Option schreibgeschützt deaktivieren.

  • Frage 7: Kann ich die Bezeichnungen der Fahrtarten oder Bootstypen ändern bzw. weitere hinzufügen?
    Antwort: Ja! Alle Bezeichnungen sind in der Datei <efa-Verzeichnis>\cfg\bezeichnungen.cfg gespeichert. Um die Bezeichnungen zu ändern oder neue hinzuzufügen, öffne diese Datei mit einem beliebigen Text-Editor (z.B. Notepad). Nach der Arbeit speichere diese Datei ab und starte efa neu. Es erscheint zunächst eine Meldung, daß die Datei von einem externen Programm verändert wurde. Nach dem Bestätigen dieser Meldung wird die Prüfsumme angepaßt, so daß efa beim nächsten Start die Datei ohne Warnung akzeptiert.
    Beim Ändern von Bezeichnungen sollten keine Bezeichnungen geändert werden, die bereits für Fahrten oder Boote benutzt werden, da efa diese sonst nicht mehr zuordnen kann (ggf. sollten nach einem Ändern der Bezeichnungen die betreffenden Fahrten oder Boote angepaßt werden). Gleiches gilt für das Löschen von benutzten Bezeichnungen.
    Beim Hinzufügen von weiteren Bezeichnungen müssen folgende Präfixe benutzt werden:
    BART= für Bootsarten,
    BANZAHL= für die Anzahl der Ruderplätze im Boot,
    BRIGGER= für die Riggerung von Booten,
    BSTM= für die Eigenschaft mit/ohne Steuermann und
    FAHRT= für Fahrtarten.

  • Frage 8: Welche Java-Version sollte für efa benutzt werden?
    Antwort: efa benötigt mindestens Java Version 1.4.0. Für optimale Funktionalität sollte immer eine möglichst aktuelle Java-Version eingesetzt werden. Zur Zeit empfehle ich, mindests Java Version 5, besser sogar Version 6, einzusetzen.
    Bei der Installation einer neuen Java-Version gehe folgendermaßen vor: Deinstalliere zuerst die aktuell benutzte Version über die Software-Deinstallationsfunktion Deines Betriebssystems. Downloade und installiere anschließend die neue Version wie hier beschrieben.

  • Frage 9: Wie kann ich den Ausdruck von Statistiken verbessern?
    Antwort: Statistiken, die in efa für die Ausgabeart im Programm (Grafik) erstellt und im internen Browser angezeigt werden, können über die Druckfunktion dort ausgedruckt werden. Diese Druckfunktion ist jedoch sehr rudimentär. Insbesondere kommt es vor, daß am Seitenende oder -anfang halbe Zeilen gedruckt werden oder die Schrift sehr klein ist. Für diese Probleme gibt es jedoch Lösungsmöglichkeiten:
    • Ausgabe in einem externen Browser:
      Wenn als Ausgabeart im Browser gewählt wird, so zeigt efa die Statistik in einem externen Webbrowser an. Dieser verfügt über deutlich umfangreichere und bessere Möglichkeiten, die Statistik auszudrucken. Insbesondere drucken die meisten Webbrowser keine halben Zeilen. In den meisten Webbrowsers läßt sich darüber hinaus auch die Schriftgröße erhöhen. Für besonders breite Tabellen empfielt sich ein Ausdruck im Querformat, der sich in den meisten Webbrowsern unter dem Menüpunkt Seite einrichten (oder ähnlich) einstellen läßt.
    • Größere Schrift im internen Browser:
      Auch im internen Browser von efa läßt sich die Schrift vergrößern. Hierzu muß man wissen, daß efa den Ausdruck so skaliert, daß der im Browser angezeigte Bereich der Seitenbreite entspricht. Um im Ausdruck eine größere Darstellung zu erhalten, muß vor dem Ausdruck lediglich das Browserfenster schmaler gemacht werden (mit der Maus am rechten Fensterrand anfassen und schmaler ziehen). Für besonders breite Tabellen kann darüber hinaus auch in efa für den Ausdruck über den Button Seite einrichten (rechts neben dem Drucken-Button) das Querformat ausgewählt werden.

  • Frage 10: Wie kann ich Statistiken in Excel importieren?
    Antwort: Um eine Statistik in Excel zu importieren, erstelle in efa eine CSV-Datei (comma separated values) und öffne diese Datei in Excel. Dieser Vorgang ist im folgenden für die deutsche Version von MS Excel 2002 ausführlich beschrieben. Bitte beachte, daß sich die einzelnen Schritte in anderen Excel-Versionen etwas unterscheiden können.
    • Wähle im Statistikfenster von efa als Ausgabeart die Option als CSV-Datei. Im Feld Ausgabe in Datei erscheint der Dateiname, unter dem efa diese Datei erzeugen wird.
    • Erstelle nun wie gewohnt durch Klicken von Statistik erstellen die gewünschte Statistik.
    • Öffne die erstellte CSV-Datei nun in Excel. Am einfachsten ist es dazu, im Explorer in das Verzeichnis zu wechseln, in dem efa die CSV-Datei erzeugt hat (standardmäßig c:\programme\efa\daten), und auf die Datei doppelzuklicken. Sollte hierbei Excel nicht starten, muß statt dessen Excel manuell gestartet werden und die CSV-Datei über den Punkt Datei öffnen im Datei-Menü ausgewählt werden.
    • Nach dem Öffnen der Datei befinden sich alle Zeilen der Datei in der ersten Spalte (Spalte A). Markiere nun die gesamte Spalte durch Klicken auf den Spaltenkopf mit der Auffschrift A.
    • Wähle nun im Menü Daten den Menüpunkt Text in Spalten ...
    • Wähle im sich öffnenden Fenster die Option Getrennt und klicke Next.
    • Kreuze nun unter Trennzeichen die Option Andere an (alle anderen Möglichkeiten deaktivieren) und gib in das Feld hinter Andere das Zeichen | ein (der Senkrechtstrich wird über die Tastenkombination <Alt Gr>-<kleiner-größer> eingegeben; die <kleiner-größer>-Taste ist links neben dem <Y>). Klicke nun Weiter.
    • Damit auch Kommazahlen korrekt importiert werden, klicke nun Weitere ... und gib in das Feld Dezimaltrennzeichen das Zeichen . (Punkt) ein und in das Feld 1000er-Trennzeichen das Zeichen , (Komma) ein. Klicke anschließend OK.
    • Klicke nun auf Fertig stellen.

  • Frage 11: Kann efa auch unter Linux eingesetzt werden
    Antwort: Selbstverständlich! efa läuft unter allen Betriebssystemen, für die Java verfügbar ist. Linux wird von efa voll unterstützt und kann vom Entwickler wärmstens empfohlen werden!
    Beim Einsatz von efa unter Linux sollte die ZIP-Datei (nicht die EXE-Datei) von efa heruntergeladen werden.

  • Frage 12: Wie erstelle ich eine Meldedatei?
    Antwort: Um eine Meldedatei für einen Kilometerwettbewerb zu erstellen, wähle im Statistikfenster auf der Registerkarte Wettbewerbe in der Liste Ausgabeart den Punkt als Meldedatei aus. Das Erstellen und Einsenden von Meldedateien im Detail ist hier beschrieben.

  • Frage 13: Wie kann ich Informationen über die Erfüllung von Wettbewerben in normalen Kilometerlisten ausgeben?
    Antwort: Um in einer normalen Kilometerliste auch auszugeben, wer bereits die Bedingungen für das DRV-Fahrtenabzeichen oder die Kilometerwettbewerbe des LRV-Berlin erfüllt hat bzw. wie viele Kilometer noch fehlen, wähle im Statistikfenster alle Einstellungen wie für eine normale Kilometerliste üblich. Klicke dann auf erweiterte Ausgabe... und gehe dort zur Registerkarte Wettbewerbe. Wähle hier nun die Wettbewerbe und -jahre aus, die als zusätzliche Spalten in der Kilometerliste ausgegeben werden sollen und klicke anschließend OK.

  • Frage 14: Wie kann ich alle Fahrten eines einzelnen Mitglieds ausgeben?
    Antwort: Um alle Fahrten eines einzelnen Mitglieds auszugeben, öffne das Statistikfenster und wähle als Statistikart die Statistik Fahrtenbuch aus. Um die Ausgabe auf die Fahrten eines einzelnen Mitglieds zu beschränken, trage in das Feld nur Name den Namen des Mitglieds ein. Klicke anschließend Statistik erstellen.

  • Frage 15: Wie kann ich zwei Fahrtenbücher zusammenfügen?
    Antwort: Um zwei Fahrtenbücher zusammenzufügen, kann die Import-Funktion von efa genutzt werden. Sie importiert alle Daten eines anzugebenden Fahrtenbuchs in das gerade geöffnete Fahrtenbuch. Dabei ist es möglich, auch nur eine Auswahl von Fahrten zu importieren oder bei kollidierenden Laufenden Nummern entsprechend zu reagieren (z.B. die Laufenden Nummern des zu importierenden Fahrtenbuchs hinten anzuhängen). Außerdem können über diese Funktion auch neue Einträge aus den Mitglieder-, Boots- und Ziellisten des zu importierenden Fahrtenbuchs mit übernommen werden.
    Um Fahrten eines anderen Fahrtenbuchs in das aktuell geöffnete Fahrtenbuch zu importieren, wähle in efa aus dem Datei-Menü den Punkt Fahrtenbuch importieren. Eine Erklärung der dort befindlichen Optionen liefert die Online-Hilfe.

  • Frage 16: Wie gebe ich in efa Fahrten ein, bei denen auf der Fahrt die Besatzung gewechselt hat?
    Antwort: Solche Fahrten können am besten in efa eingegeben werden, indem sie in einzelne Teilfahrten aufgeteilt werden.
    Beispiel (eintägige Fahrt):
    LfdNr: 123 Boot: B Ruderer: V,W,X,Y,Z Strecke: 50 Km, wobei Y und Z sich abgewechselt haben (einer 20, der andere 30 Km).
    Eingegeben werden:
    LfdNr: 123 Boot: B Ruderer: V,W,X Strecke: 50 Km
    LfdNr: 123A Boot: B Ruderer: Y Strecke: 20 Km
    LfdNr: 123B Boot: B Ruderer: Z Strecke: 30 Km
    Beispiel (mehrtägige Fahrt):
    • Wenn sich die einzelnen Ruderer tageweise abgewechselt haben, dann kann die Fahrt einfach in Form von Etappen eingegeben werden. Die einzelnen Etappen werden über den Punkt Fahrtart dem Namen der Mehrtagesfahrt zugeordnen, zu der sie gehören.
    • Wenn sich die Ruderer auch innerhalb einer Etappe abgewechselt haben: In diesem Fall wird die Fahrt in Form von Etappen eingegeben und für die betreffende Etappe wie bei eintägigen Fahrten verfahren. Dabei sollte darauf geachtet werden, daß im Feld Ziel für alle Einträge derselben Etappe die gleiche Bezeichnung steht, damit efa die Einträge als gemeinsame Etappe erkennt. Beim Auswerten der Wanderruderstatistik kommt dann eine Warnung, daß die Etappe mit unterschiedlichen Entfernungen vorkommt, aber diese Warnung kann dann ignoriert werden.
    Weitere Variante für eintägige Fahrten: Oben genannte Variante verfälscht die Bootskilometer, da dort Boot B die Strecke (aus Sicht von efa) zweimal gefahren ist. Wer Wert darauf legt, daß auch die Bootskilometer korrekt sind, kann folgendermaßen vorgehen:
    LfdNr: 123A Boot B: Ruderer V,W,X,Y Strecke: 20 Km
    LfdNr: 123B Boot B: Ruderer V,W,X,Z Strecke: 30 Km
    Bei dieser Art des Eintragens werden die beiden Teilstrecken vor und nach dem Wechsel mit der jeweils kompletten Mannschaft erfaßt. Damit nun aber den Ruderern V,W, und X eine 50 Km lange Tagesetappe für die Wanderruderwettbewerbe anerkannt wird, müssen die beiden Einträge 123A und 123B über den Punkt Fahrtart zu einer "Mehrtagesfahrt" mit einem Tag Länge zusammengefaßt werden.

  • Frage 17: Unser Verein hat zwei Bootshäuser, die unabhängig voneinander jeweils ein eigenes Fahrtenbuch führen. Wie kann ich die Fahrtenbücher beider Bootshäuser gemeinsam auswerten?
    Antwort: Wenn ein Verein zwei Bootshäuser hat und damit zwei verschiedene Fahrtenbücher führen muß, gibt es verschiedene Möglichkeiten, wie diese beiden Fahrtenbücher gemeinsam ausgewertet werden können. Zunächst sollten beide Fahrtenbücher unterschiedliche Dateinamen tragen, etwa ein angehängtes "a" und "b" für die beiden Standorte, z.B. "2008a.efb" und "2008b.efb".

    Die einfachste und eleganteste Variante, beide Fahrtenbücher gemeinsam auszuwerten, ist die "fahrtenbuchübergreifende" Auswertung in efa. Bei dieser Auswertungsvariante berücksichtigt efa alle miteinander verknüpften Fahrtenbücher.
    Standardmäßig verknüpft efa beim erstellen eines neuen Fahrtenbuchs über die Funktion "Fahrtenbuch fortsetzen" automatisch aktuellen und neues Fahrtenbuch miteinander (etwa beim Beginn eines Fahrtenbuches für das neue Jahr). Es ergibt sich damit eine Kette von Fahrtenbüchern, etwa:
    2006.efb <-> 2007.efb <-> 2008.efb
    Wenn das Fahrtenbuch 2008.efb geöffnet ist und in der Statistikauswertung die Option "fahrtenbuchübergreifend" aktiviert ist, verfolgt efa die Kette der Verknüpfungen bis zum Anfang und wertet alle Fahrtenbücher in dieser Kette aus. Über Angabe von Daten für den Auswertungszeitraum kann die Auswertung auf ein Jahr eingeschränkt werden.
    Die Verkfüpfungen zwischen Fahrtenbüchern können vom Benutzer nach Belieben verändert werden. Dies ist möglich im Admin-Modus über den Punkt "Fahrtenbuch-Vollzugriff" oder im "normalen" efa. Dort gelangt man über das Menü "Administration" und die Punkte "Fahrtenbuch" - "Einstellungen zum Fahrtenbuch" zu den Verknüpfungen "vorherige Fahrtenbuchdatei" und "nächste Fahrtenbuchdatei". Wichtig ist, daß die Verknüpfungen immer in beiden Richtungen erstellt werden, d.h. beispielsweise: Wenn im 2008er Fahrtenbuch als "vorherige Fahrtenbuchdatei" das Fahrtenbuch "2007.efb" eingetragen ist, dann muß auch im 2007er Fahrtenbuch als "nächste Fahrtenbuchdatei" das Fahrtenbuch "2008.efb" eingetragen sein. Das erste und letze Fahrtenbuch in der Verknüpfungskette haben logischerweise keine vorherige bzw. nächste Fahrtenbuchdatei.
    Dies kann man nun benutzen, um die Fahrtenbücher mehrerer Bootshäuser gemeinsam auszuwerten. Man kopiert alle Fahrtenbücher beider Bootshäuser (inkl. der Mitglieder-, Boots- und Ziellisten, also die Dateien *.efb, *.efbm, *.efbb und *.efbz) auf einen PC und ändert die Verknüpfungen zwischen ihnen so ab, daß die Verknüpfungskette wie folgt aussieht:
    2006a.efb <-> 2006b.efb <-> 2007a.efb <-> 2007b.efb <-> 2008a.efb <-> 2008b.efb
    Über die Option "fahrtenbuchübergreifend" können nun alle Fahrtenbücher ausgewertet werden. Wenn man nur an dem Jahr 2008 interessiert ist, schränkt man die Auswertung auf "1.1.2008" bis "31.12.2008" ein.
    Eine weitere Optimierung für die häufige Auswertung einer über die Jahre sehr lang gewordenen Kette von Fahrtenbüchern ist die Einschränkung der Auswertung auf zwei Fahrtenbücher aus dieser Kette: Im Statistikfenster unter dem Punkt "erweiterte Ausgabe ..." auf der Registerkarte "Weitere" können unter "nur die folgenden Fahrtenbücher auswerten" zwei Fahrtenbücher aus der Kette angegeben werden, etwa "2008a.efb" und "2008b.efb". In diesem Fall berücksichtigt die fahrtenbuchübergreifende Auswertung nur diese beiden Fahrtenbücher. Wer will, kann diese Einstellung über den Punkt "Statistikeinstellungen speichern" abspeichern, um sie nicht für jede Auswertung erneut vornehmen zu müssen.
    Der Vorteil dieser Variante besteht darin, daß die Fahrtenbücher selbst nicht verändert werden müssen (abgesehen von den Verknüpfungen) und die Fahrtenbücher beider Bootshäuser weiterhin eigene Boots- und Ziellisten haben, so daß es zu keinen Kollisionen von gleichen Namen kommen kann (etwa wenn das gleiche Ziel in beiden Bootshäusern in der Zielliste steht, jedoch mit unterschiedlichen Entfernungen).

    Eine zweite Variante besteht darin, die Fahrtenbücher beider Bootshäuser in ein gemeinsames Fahrtenbuch zu importieren. Zum Beispiel erstellt man auf dem Auswertungs-PC ein Fahrtenbuch 2008.efb und importiert über die Funktion "Fahrtenbuch importieren" im "Datei"-Menü alle Fahrten der Fahrtenbücher 2008a.efb und 2008b.efb in dieses gemeinsame Fahrtenbuch. Damit die Laufenden Nummern beider Fahrtenbücher nicht kollidieren, kann unter "Zu allen importierten Fahrten: ..." entweder eine Zahl zu der LfdNr hinzuaddiert oder ein Buchstabe angefügt werden. Beispielsweise könnte man zu den Fahrten des einen Bootshauses immer ein 10000 hinzuaddieren und zu denen des zweiten Bootshauses eine 20000. So kann man auch im Nachhinein die Fahrten noch unterscheiden.
    Der Nachteil dieser Variante ist jedoch, daß bei gleichnamigen Zielen oder Booten immer nur ein Ziel bzw. Boot in der jeweiligen Liste stehen kann. Die dort hinterlegten Informationen sind u.U. nur für die Fahrten im einen Bootshaus korrekt, für die des anderen Bootshaus jedoch falsch, falls es bspw. zwei gleichnamige Boote unterschiedlichen Typs gibt, oder von beiden Bootshäusern dasselbe Ziel mit unterschiedlicher Entfernung erreichbar ist.

  • Frage 18: Wie kann ich verhindern, daß Mitglieder eine begonnene Fahrt abbrechen, um zu verschleiern, daß sie als letztes mit dem Boot gerudert sind, wenn sie einen Bootsschaden verursacht haben?
    Antwort: Einstellungen, daß Fahrten nicht abgebrochen werden können oder immer im Fahrtenbuch landen, gibt es nicht.

    Allerdings protokolliert efa in der Logdatei alle relevanten Aktionen durch die Mitglieder, u.a. auch das Beginnen, Beenden und Abbrechen von Fahrten. Wenn Du im Admin-Modus in die Logdatei schaust (oder direkt im Texteditor efadirekt.log im efa-Verzeichnis öffnest), siehst Du alle Aktionen. Suche nach "Fahrtbeginn", "Fahrtende" oder "Fahrtabbruch". Damit ist sichergestellt, daß alle Fahrten nachvollzogen werden können.

    Wie Du angemerkt hast, ist ein Löschen von Fahrten in einem Papier-Fahrtenbuch nicht möglich. Damit efa als Nachweis gegenüber Polizei und Versicherungen Bestand haben kann, muß es ebenfalls sicherstellen, daß Fahrten nicht durch Mitglieder gelöscht werden können. Für einmal beendete Fahrten gilt dies ohnehin, und über die Logdatei gilt dies auch für das Abbrechen von Fahrten.

    Wenn also ein Mitglied einen Bootsschaden verursacht und anschließend die Fahrt abbricht, damit niemand sieht, wer zuletzt mit dem Boot gerudert ist, kannst Du das als Administrator anhand der Logdatei leicht herausfinden und das Mitglied zur Rede stellen. Das Abbrechen einer Fahrt, die tatsächlich stattgefunden hat und sogar zu einem Schaden geführt hat, ist ein schwerer Verstoß des Mitglieds gegen die Nachweispflicht von Fahrten im Fahrtenbuch.

  • Frage 19: Ich erhalte Warnungen Der aktuelle Speicherverbrauch ist relativ hoch. Was kann ich tun?
    Antwort: Diese Warnung bedeutet, daß der frei Arbeitsspeicher, der efa zur Verfügung steht, knapp wird. Ursache können sehr große Fahrtenbücher (viele tausend Einträge), das Aufrufen umfangreicher Statistiken oder möglicherweise auch Fehler in efa oder Java sein.

    Generell sollte zur Vermeidung von Speicherproblemen efa auch im Bootshaus regelmäßig (etwa einmal täglich) neu gestartet werden. Du kannst einen automatischen Neustart von efa in der Konfiguration unter Administration - Allgemeine Einstellungen - efa im Bootshaus unter efa automatisch um hh:mm Uhr neu starten durch Eintragen einer Uhrzeit zum Neustart aktivieren.

    Sollten trotz regelmäßiger Neustarts dennoch Warnungen wegen hohem Speicherverbrauch auftreten, so liegt dies vermutlich an der Größe des Fahrtenbuchs oder dem Umfang der Statistiken. In diesem Fall kann der Arbeitsspeicher, der efa von Java zur Verfügung gestellt wird, heraufgesetzt werden. Dies kann insbesondere bei knappem Speicher auch zu deutlichen Geschwindigkeitssteigerungen führen. Standardmäßig verwendet efa maximal 64 MB Arbeitsspeicher, wovon 16 MB für die sogenannte New Generation reserviert ist. Diese beiden Werte können durch die Variablen EFA_JAVA_HEAP (Standardwert: 64m) und EFA_NEW_SIZE (Standardwert: 16m) gesetzt werden und sollten immer gemeinsam erhöht werden. Dabei ist zu beachten, daß der Computer natürlich über genügend physikalischen Arbeitsspeicher verfügen sollte.
    Um die Variablen EFA_JAVA_HEAP und EFA_NEW_SIZE anzupassen, erstelle bitte in einem Texteditor eine Datei namens javaheap.bat (Windows) bzw. java.heap (Linux, Solaris) in dem efa-Installationsverzeichnis (unter Windows standardmäßig c:\programme\efa; dort, wo auch die Startdateien efadirekt.bat bzw. efadirekt.sh liegen). Um den efa zur Verfügung stehenden Speicher zu verdoppeln, erstelle bitte eine solche Datei javaheap.bat mit folgendem Inhalt:
    SET EFA_JAVA_HEAP=128m
    SET EFA_NEW_SIZE=32m
    Um dasselbe unter Linux bzw. Solaris zu erreichen, erstelle bitte eine Datei mit Namen java.heap und folgendem Inhalt:
    EFA_JAVA_HEAP=128m
    EFA_NEW_SIZE=32m
    Anschließend muß efa neu gestartet werden.
